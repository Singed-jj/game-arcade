Part B. 코드로 만드는 이펙트 (개발자용)

게임의 "기분 좋은 느낌"은 이미지가 아니라 이펙트가 만든다.
아래 이펙트는 전부 Canvas 2D 또는 CSS 애니메이션으로 구현한다. 별도 이미지 에셋 불필요.

게임 엔진 참고: 웹뷰 기반이면 PixiJS (2D WebGL 렌더러)를 추천. Canvas 2D보다 파티클 성능이 좋고, 트윈/이징 라이브러리가 내장되어 있음. 아래 코드는 순수 Canvas/CSS 기준이지만 PixiJS로 그대로 치환 가능.

B-1. 블록 터짐 이펙트

3개 이상 매칭된 블록이 사라질 때. 게임에서 가장 자주 발생하는 이펙트.

동작 순서 (0.3초 안에 전부 일어남)

시간무슨 일이 일어나나
0ms매칭 판정. 해당 블록들에 "터질 예정" 표시
0~100ms블록이 1.2배로 커졌다가 빠르게 0으로 축소 (scale: 1→1.2→0). 동시에 투명도 1→0
50ms블록 중심에서 하얀 원형 flash가 한 번 번쩍 (0.15초간 나타났다 사라짐)
80ms블록 위치에서 파티클 6~8개 방출. 블록 색깔의 작은 원/사각형이 랜덤 방향으로 튀어나감
100ms블록 위치에 "+점수" 텍스트가 나타남. 위로 올라가면서 사라짐 (0.5초)
300ms파티클 소멸. 이펙트 완전 종료
코드 — 블록 축소 애니메이션 (CSS) /* 매칭된 블록에 이 클래스를 추가하면 터지는 애니메이션 실행 */ @keyframes block-pop { 0% { transform: scale(1); opacity: 1; } 30% { transform: scale(1.2); opacity: 1; } /* 살짝 커짐 (팽창) */ 100% { transform: scale(0); opacity: 0; } /* 사라짐 */ } .block.matched { animation: block-pop 0.2s ease-in forwards; /* ease-in = 처음엔 천천히, 끝에 빠르게 → 터지는 느낌 */ }
코드 — 파티클 시스템 (Canvas 2D 의사코드) // 블록 1개가 터질 때 호출 function spawnParticles(x, y, color) { const count = 6 + Math.random() * 3; // 6~8개 for (let i = 0; i < count; i++) { particles.push({ x: x, // 블록 중심 좌표 y: y, vx: (Math.random() - 0.5) * 8, // 속도: 좌우 랜덤 (-4 ~ +4) vy: (Math.random() - 0.5) * 8, // 속도: 상하 랜덤 size: 3 + Math.random() * 4, // 크기: 3~7px color: color, // 블록 색깔 life: 1.0, // 수명 (1.0 → 0.0으로 감소) gravity: 0.15, // 중력: 아래로 끌림 }); } } // 매 프레임마다 호출 (requestAnimationFrame 안에서) function updateParticles(dt) { for (const p of particles) { p.x += p.vx; p.y += p.vy; p.vy += p.gravity; // 중력 적용 → 포물선 궤적 p.life -= 0.04; // 수명 감소 → 0 되면 제거 p.size *= 0.97; // 점점 작아짐 } particles = particles.filter(p => p.life > 0); } // 그리기 function drawParticles(ctx) { for (const p of particles) { ctx.globalAlpha = p.life; // 수명에 따라 투명해짐 ctx.fillStyle = p.color; ctx.fillRect(p.x - p.size/2, p.y - p.size/2, p.size, p.size); // 사각형 대신 원으로 하려면 ctx.arc() 사용 } ctx.globalAlpha = 1; } // 블록별 파티클 색상 매핑 const BLOCK_COLORS = { chicken: '#F5A623', // 금색 cola: '#D0021B', // 빨강 fries: '#F8E71C', // 노랑 burger: '#8B572A', // 갈색 pizza: '#FF6B35', // 주황 };
코드 — 하얀 flash 효과 (Canvas 2D) // 블록이 터지는 순간, 블록 위치에 원형 flash function spawnFlash(x, y) { flashes.push({ x: x, y: y, radius: 5, // 시작 크기 maxRadius: 30, // 최대 크기 life: 1.0, }); } function drawFlashes(ctx) { for (const f of flashes) { const r = f.maxRadius * (1 - f.life) + f.radius; // 점점 커짐 ctx.globalAlpha = f.life * 0.6; // 점점 투명해짐 ctx.fillStyle = 'white'; ctx.beginPath(); ctx.arc(f.x, f.y, r, 0, Math.PI * 2); ctx.fill(); f.life -= 0.08; // ~12프레임(0.2초)이면 사라짐 } ctx.globalAlpha = 1; flashes = flashes.filter(f => f.life > 0); }
코드 — 점수 텍스트 떠오르기 (CSS) @keyframes score-float { 0% { transform: translateY(0); opacity: 1; } 100% { transform: translateY(-40px); opacity: 0; } } .score-popup { position: absolute; font-size: 16px; font-weight: 900; color: white; text-shadow: 0 2px 4px rgba(0,0,0,0.5); animation: score-float 0.5s ease-out forwards; pointer-events: none; /* 클릭 방해 안 함 */ } /* JS에서: 블록 터진 위치에 div 생성 → 0.5초 후 제거 */

B-2. 캐스케이드(연쇄) 이펙트

블록이 떨어진 뒤 또 매칭이 일어나는 것. 연쇄가 길수록 더 화려해야 한다.

연쇄 횟수별 연출 단계

연쇄텍스트글씨 크기색상추가 효과
x1(없음)--기본 터짐 이펙트만
x2"Good!"24px흰색화면 흔들림 약하게
x3"Great!!"32px노랑화면 흔들림 + 배경 flash
x4"Amazing!!!"40px주황흔들림 강하게 + 배경 밝아짐
x5+"INCREDIBLE!!!!"48px무지개최대 이펙트
코드 — 화면 흔들림 (Screen Shake) // 게임 보드 전체를 감싸는 컨테이너에 적용 let shakeIntensity = 0; let shakeDuration = 0; function triggerShake(intensity, duration) { // intensity: 흔들림 세기 (px). 연쇄 x2=2, x3=4, x4=6, x5+=8 // duration: 지속 시간 (ms). 보통 200~300 shakeIntensity = intensity; shakeDuration = duration; } function updateShake() { if (shakeDuration <= 0) { boardContainer.style.transform = 'translate(0, 0)'; return; } // 랜덤 방향으로 흔들기 const dx = (Math.random() - 0.5) * shakeIntensity * 2; const dy = (Math.random() - 0.5) * shakeIntensity * 2; boardContainer.style.transform = `translate(${dx}px, ${dy}px)`; shakeDuration -= 16; // 매 프레임 감소 (60fps 기준 ~16ms) shakeIntensity *= 0.9; // 점점 약해짐 } // 사용 예시: // 연쇄 x2일 때: triggerShake(2, 200) // 연쇄 x4일 때: triggerShake(6, 300)
코드 — 연쇄 텍스트 (CSS) @keyframes cascade-text { 0% { transform: scale(0.3); opacity: 0; } 40% { transform: scale(1.3); opacity: 1; } /* 크게 튀어나옴 */ 60% { transform: scale(1.0); opacity: 1; } /* 살짝 줄어듬 */ 100% { transform: scale(1.0); opacity: 0; } /* 사라짐 */ } .cascade-text { position: absolute; top: 50%; left: 50%; transform-origin: center; font-weight: 900; text-shadow: 0 3px 8px rgba(0,0,0,0.4); animation: cascade-text 0.8s ease-out forwards; pointer-events: none; z-index: 100; } /* JS에서: 보드 중앙에 div 생성, 연쇄 횟수에 따라 fontSize / color / shakeIntensity 를 다르게 세팅 */
코드 — 배경 flash (연쇄 x3+) /* 보드 위에 겹치는 반투명 오버레이 */ @keyframes bg-flash { 0% { opacity: 0.3; } 100% { opacity: 0; } } .bg-flash-overlay { position: absolute; top: 0; left: 0; right: 0; bottom: 0; background: white; pointer-events: none; animation: bg-flash 0.2s ease-out forwards; } /* JS에서: 연쇄 x3 이상이면 오버레이 div 생성 → 0.2초 후 제거 */

B-3. 블록 낙하 애니메이션

터진 자리에 위 블록들이 떨어지는 것. 자연스러운 "쿵" 느낌이 중요.

코드 — 낙하 + 바운스 (CSS) @keyframes block-drop { 0% { transform: translateY(-{distance}px); } 70% { transform: translateY(0px); } /* 착지 */ 80% { transform: translateY(-4px); } /* 살짝 위로 튀어오름 */ 90% { transform: translateY(0px); } /* 다시 내려옴 */ 95% { transform: translateY(-1px); } /* 아주 살짝 */ 100% { transform: translateY(0px); } /* 최종 위치 */ } /* {distance} = 떨어지는 칸 수 × 셀 높이(px) 예: 2칸 떨어지면 distance = 2 * 64 = 128px duration: 떨어지는 거리에 비례. 1칸=0.15s, 2칸=0.25s, 3칸=0.3s → 너무 느리면 답답하고, 너무 빠르면 연쇄를 못 봄 */ .block.dropping { animation: block-drop 0.25s cubic-bezier(0.34, 1.56, 0.64, 1) forwards; /* cubic-bezier(0.34, 1.56, ...) = 살짝 오버슈트하는 커브 → 바운스 느낌 */ }

B-4. 도구 이펙트 3종

도구는 뽑기나 보상으로 얻는 귀한 것. 일반 매칭보다 2배 화려하게 만들어야 "쓴 보람"이 있다.

로켓 — 한 줄 전체 제거

시간무슨 일이 일어나나
0ms로켓 아이콘이 해당 셀에 나타남 (scale 0→1, 바운스)
200ms로켓이 빛줄기(beam)를 남기며 가로 or 세로로 이동. 이동 속도 = 0.3초에 보드 끝까지
200~500ms빛줄기가 지나간 자리의 블록들이 순서대로 터짐 (왼→오 or 위→아래). 한 칸당 30ms 간격
500ms모든 블록 터짐 완료. 파티클 잔여
코드 — 로켓 빛줄기 (Canvas 2D) // 로켓이 지나간 자리에 빛줄기 그리기 function drawRocketBeam(ctx, startX, startY, endX, endY, progress) { // progress: 0→1 (로켓 이동 진행도) const currentX = startX + (endX - startX) * progress; const currentY = startY + (endY - startY) * progress; // 빛줄기 (반투명 직사각형) const beamWidth = 12; ctx.globalAlpha = 0.6; const gradient = ctx.createLinearGradient(startX, startY, currentX, currentY); gradient.addColorStop(0, 'rgba(255, 200, 50, 0)'); // 시작점: 투명 gradient.addColorStop(0.7, 'rgba(255, 200, 50, 0.8)'); // 중간: 밝음 gradient.addColorStop(1, 'rgba(255, 255, 255, 1)'); // 끝: 흰색 ctx.fillStyle = gradient; // 가로 로켓이면: ctx.fillRect(startX, startY - beamWidth/2, currentX - startX, beamWidth); ctx.globalAlpha = 1; } // 로켓 아이콘은 A-2에서 만든 PNG를 currentX, currentY에 그리면 됨 // ctx.drawImage(rocketImg, currentX - 32, currentY - 32, 64, 64);

폭탄 — 3×3 영역 제거

시간무슨 일이 일어나나
0ms폭탄 아이콘이 해당 셀에 나타남
0~300ms폭탄이 좌우로 빠르게 흔들림 (틱틱틱 느낌). 점점 빨라짐
300ms폭발! 폭탄 사라짐 + 충격파 원이 중심에서 퍼져나감
300~500ms충격파가 퍼지면서 3×3 영역의 블록들이 동시에 터짐. 화면 흔들림(강)

C-2. 햅틱(진동) 피드백

소리 없이 진동만 있어도 "터지는 느낌"이 난다. 소리 + 진동이 같이 나오면 쾌감이 2배.
웹뷰에서는 navigator.vibrate() API를 쓴다. iOS Safari는 미지원이라 네이티브 브릿지가 필요.

코드 — 햅틱 구현 (웹뷰) // === 방법 1: Web Vibration API (Android 웹뷰에서 동작) === function haptic(pattern) { if (navigator.vibrate) { navigator.vibrate(pattern); } } // 사용 예시: haptic(15); // 블록 매칭: 15ms 짧은 진동 haptic([15, 30, 25]); // 연쇄 x3: 진동-쉼-진동 패턴 haptic([10, 10, 10, 10, 10, 10, 50]); // 로켓: 짧은 진동 반복 후 강한 진동 // 숫자 배열 = [진동ms, 쉼ms, 진동ms, 쉼ms, ...] // [15, 30, 25] = 15ms 진동 → 30ms 쉼 → 25ms 진동 // === 방법 2: iOS 네이티브 브릿지 (iOS Safari는 vibrate 미지원) === // 네이티브 앱에서 웹뷰 메시지를 받아서 UIImpactFeedbackGenerator 호출 // 웹뷰 → 네이티브: function hapticNative(type) { // type: 'light', 'medium', 'heavy', 'soft', 'rigid' window.webkit?.messageHandlers?.haptic?.postMessage(type); // 또는 Android: AndroidBridge.haptic(type); } // 네이티브 (Swift) 쪽: // func handleHaptic(_ type: String) { // switch type { // case "light": UIImpactFeedbackGenerator(style: .light).impactOccurred() // case "medium": UIImpactFeedbackGenerator(style: .medium).impactOccurred() // case "heavy": UIImpactFeedbackGenerator(style: .heavy).impactOccurred() // case "soft": UIImpactFeedbackGenerator(style: .soft).impactOccurred() // case "rigid": UIImpactFeedbackGenerator(style: .rigid).impactOccurred() // default: break // } // } // === 매핑 === // iOS에서는 패턴 대신 강도 타입으로 호출: const HAPTIC_MAP = { swap: 'light', // 블록 스왑 match: 'medium', // 3매칭 cascade2: 'medium', // 연쇄 x2 cascade3: 'heavy', // 연쇄 x3 cascade4: 'rigid', // 연쇄 x4+ rocket: 'rigid', // 로켓 bomb: 'heavy', // 폭탄 rainbow: 'heavy', // 무지개 clear: 'heavy', // 클리어 fail: 'medium', // 실패 star: 'medium', // 별 획득 wrongSwap: 'soft', // 잘못된 스왑 };
핵심 원칙:
• 짧을수록 좋다. 진동이 길면 불쾌하다. 대부분 10~30ms면 충분.
• 소리와 동시에. pop 사운드가 나는 바로 그 순간에 진동도 같이 나와야 한다.
• 연쇄는 빌드업. 연쇄가 길어질수록 진동이 강해지고 빈번해져야 "와 이거 개쩐다" 느낌이 남.
• 도구는 패턴이 있어야. 일반 매칭과 확실히 다른 진동 패턴이어야 "특별한 걸 썼다" 느낌.
• iOS는 네이티브 브릿지 필수. Web Vibration API는 iOS Safari에서 안 됨. 네이티브 앱에서 햅틱 엔진을 호출하는 브릿지를 만들어야 함.
