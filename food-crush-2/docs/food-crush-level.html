푸드크러시 — 레벨 난이도 설계

변수 2개로 무한 레벨을 만드는 법 · 시뮬레이터 포함

1. 한 줄 요약

레벨 난이도는 딱 2가지로 정해진다:
몇 번 움직일 수 있는지 + 몇 종류의 블록을 모아야 하는지

퍼즐판은 매번 랜덤으로 섞인다. 같은 레벨을 두 번 해도 블록 배치가 다르다.

그래서 난이도를 결정하는 건 이동 횟수와 목표 종류 수, 이 두 개뿐이다.

2. 왜 이 2개면 충분한가

변수뭘 하는 건지구현 비용
이동 횟수 난이도를 올리고 내린다 레벨 설정 숫자 1개
목표 종류 수 매 판 다른 느낌을 준다 (안 질리게) 레벨 설정 숫자 1개. UI 이미 있음
장애물 (얼음 등) 다양성을 더 줌 새 로직 + 그래픽 + 애니메이션 필요. v2
보드 모양 변형 다양성을 더 줌 레벨 에디터 필요. v2

목표 종류 수가 왜 중요하냐면 — "치킨 15개 모으기"를 10번 반복하면 3판 만에 질린다. 근데 "치킨 12개 + 콜라 8개"가 되면 어떤 블록을 먼저 터뜨릴지 전략이 생긴다. 같은 보드인데 플레이 느낌이 완전히 다르다.

3. 목표 수량은 자동으로 정해진다

목표 블록을 "몇 개" 모아야 하는지도 별도로 설계할 필요 없다. 이동 횟수에서 자동으로 나온다:

시뮬레이션 결과, 목표 종류 수에 따라 배율이 달라야 한다. 2종류 목표는 5종 블록 중 2종을 세니까 1종보다 훨씬 쉽다.

2종: 종류당 = 이동 × 1.0   |   3종: 종류당 = 이동 × 1.0
이동 16, 2종이면 → 치킨 16 + 콜라 16 (총 32개)  |  이동 11, 3종이면 → 11+11+11 (총 33개)
이동목표 2종목표 3종

18🍗 18 + 🥤 18 = 36개-
12🍗 14 + 🥤 14 = 28개🍗 12 + 🥤 12 + 🍟 12 = 36개

5. 톱니(Sawtooth) 패턴

레벨 6부터 난이도가 올라갔다 → 내려갔다를 반복한다. 핵심 2가지:

① 어려운 판 다음에 반드시 쉬운 판이 온다. 어려운 걸 2번 연속 실패하면 짜증나서 떠나니까.
② 쉬운 판도 점점 어려워진다. 숨돌리기 클리어율이 66% → 54% → 52% → 50% → 43%... 이렇게 떨어진다. 전반적으로 우하향하는 그래프.

6. 레벨 11부터는 무한 자동 생성 ∞ INFINITE

레벨 1~10은 직접 설계한다 (숫자 2개씩, 총 20개 숫자). 레벨 11부터는 공식으로 자동 생성:

자동 생성 규칙

항목규칙예시
패턴4판 1세트: 숨돌리기 → 어려움 → 숨돌리기 → 보스Lv 11(쉬움) → 12(어려움) → 13(쉬움) → 14(보스)
핵심매 세트마다 조금씩 더 어려워짐숨돌리기 클리어율: 52% → 50% → 43% → 41% → ...
숨돌리기이동 = 14 - 세트번호, 2종 목표세트1: 14이동(52%) → 세트2: 13이동(43%)
어려운 판이동 11, 목표 = 13 + 세트번호세트1: 13+13(32%) → 세트2: 14+14(23%)
보스 판이동 10, 3종 목표 = 10 + 세트번호세트1: 10+10+10(34%) → 세트2: 11+11+11(29%)
목표 블록5종 중 랜덤 선택매번 다른 조합
하한클리어율 최소 15% 이하로는 안 내려감이동 최소 9, 어느 순간 수렴

매일 새 레벨이 열리니까 "오늘은 어떤 조합일까?" 하고 들어온다. 레벨 번호가 계속 올라가니까 "나 벌써 레벨 47이야" 하는 성취감도 생긴다.

개발 포인트: 레벨 11부터는 서버에서 seed 기반으로 생성하면 저장 공간 불필요. seed = 레벨 번호 + 날짜면 모든 유저가 같은 레벨을 플레이 → 소셜 비교 가능 ("레벨 23 깼어?" "나 아직...")

9. 개발자 전달 사항

레벨 config 구조 (JSON)

{
  "level": 5,
  "moves": 17,
  "objectives": [
    { "block": "chicken", "count": 18 },
    { "block": "cola", "count": 18 }
  ]

레벨 11+는 서버에서 자동 생성. 클라이언트는 이 JSON만 받으면 어떤 레벨이든 플레이 가능.

// 무한 레벨 생성 공식 (레벨 11+)
cycle = Math.floor((level - 11) / 4)  // 0, 1, 2, ...
pos   = (level - 11) % 4              // 0, 1, 2, 3

pos 0 (숨돌리기): moves = max(10, 14 - cycle), types = 2, perType = 14 + cycle
pos 1 (어려움):   moves = max(9, 11),          types = 2, perType = 13 + cycle
pos 2 (숨돌리기): moves = max(10, 14 - cycle), types = 2, perType = 15 + cycle
pos 3 (보스):     moves = 10,                  types = 3, perType = 10 + cycle
